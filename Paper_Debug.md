# 中文摘要（初稿）

随着城市即时配送需求的不断增长，面向新能源车辆的物流调度问题逐渐呈现出任务动态到达、车辆资源有限以及补能约束复杂等特点。针对传统配送调度中难以同时兼顾路径规划、任务分配、电量管理与充电拥堵的问题，本文设计并实现了一套基于图结构与智能决策方法的新能源物流车队动态协同调度系统。系统以城市路网为基础，将道路抽象为带权图结构，结合车辆电量、载重、任务时限以及充电站排队负荷等现实约束，构建了一个面向动态任务场景的离散时间仿真框架。

在方法上，本文首先利用图结构对道路网络、配送站点、任务节点和充电站进行统一建模，并采用 Dijkstra、A* 与 RRT 等寻路算法完成车辆路径搜索与可达性判断；在调度层面，实现了最近任务优先、最大重量优先、最早截止时间优先等多种启发式策略，并进一步引入基于 Q-learning 的超启发式决策方法，通过状态离散编码与规则选择机制提升系统在复杂动态场景下的自适应能力。与此同时，本文还构建了上帝视角下的离线 MILP 优化模块，用于求解小规模场景的全局近优调度方案，并与在线动态调度结果进行对比分析。

在系统实现方面，项目采用“仿真引擎 + 后端接口 + 可视化前端”的整体架构：后端负责环境建模、调度执行、日志记录和策略运行，前端提供车辆、任务、充电站和统计指标的实时可视化展示，并支持离线求解回放、Q-learning 训练与推理等交互功能。实验结果表明，该系统能够较完整地模拟新能源物流配送中的关键约束与运行过程，不同调度策略在任务完成率、总收益、路径长度和充电负载等指标上表现出明显差异；其中，引入强化学习后的策略选择方法在复杂动态场景中展现出一定的灵活性与扩展潜力。

总体而言，本文工作完成了一个面向课程项目要求、兼具算法研究与工程实现价值的新能源物流车队动态协同调度平台，可为后续在多智能体协同、精确优化求解与更复杂城市配送场景上的深入研究提供基础。

# Introduction（中文版初稿）

## 1. 研究背景与问题提出

随着电子商务、同城即时配送和智慧物流的快速发展，城市配送系统面临着更高的时效性要求与更复杂的运行约束。与传统燃油车辆相比，新能源物流车具有低碳、节能和政策支持等优势，已逐渐成为城市物流配送的重要载体。然而，新能源车辆同时也带来了新的调度挑战：其电量有限、补能过程耗时、充电站资源有限且可能拥堵，这些因素使得原本就复杂的车辆路径规划与任务调度问题变得更加困难。

在实际物流场景中，配送任务往往并非在系统开始时一次性全部已知，而是随着时间推移不断动态出现。管理者需要在车辆数量有限、单车载重受限、电量持续消耗、任务有截止时间、充电站存在排队与负荷压力等多重约束下，实时决定应由哪一辆车执行哪一个任务、如何规划行驶路径、何时前往充电以及是否允许多辆车协同完成同一任务。因此，该问题并不是简单的最短路径计算，而是一个融合了图搜索、资源约束调度、动态决策与系统仿真的综合性问题。

基于上述背景，We aim to目标设计并实现一个完整系统平台，用于模拟和分析新能源物流场景下不同调度策略的运行效果。

## 2. 围绕课程要求的系统目标拆解

根据课程给出的《2026-大作业要求》，本项目需要完成一个能够反映新能源物流配送真实特点的调度系统。围绕这些要求，可以将系统目标拆分为以下几个层面。

### 2.1 车辆与任务的动态建模

本项目在实体层中对车辆、任务、仓库与充电站进行了统一建模：车辆对象包含当前位置、电量、电池容量、载重上限、当前负载、速度和单位里程耗电等属性；任务对象包含释放时间、截止时间、任务位置、货物重量和状态等信息。系统在仿真过程中按照时间推进动态释放任务，而不是将全部任务一次性提供给调度器，从而更贴近真实配送过程中的在线决策场景。

### 2.2 图结构道路网络与路径规划

本项目以图作为核心数据结构，将道路网络抽象为带权图，其中节点表示路网交汇点、仓库、任务点和充电站，边表示道路连接关系及其距离/通行代价。在寻路算法方面，系统实现了 Dijkstra 算法作为稳定可靠的最短路径基线，同时引入 A* 算法提升目标导向搜索效率，并加入 RRT 作为扩展性实验算法，用于展示系统在路径规划层面的可扩展能力。通过这些算法，系统能够完成车辆最短路查询、可达性判断、回仓可行性判断以及最近充电站搜索等功能。

### 2.3 评分机制与充电约束建模

本项目在仿真环境中实现了任务收益计算与过期惩罚机制，并将充电站设计为具有充电桩数、占用状态和等待队列的资源节点。系统不仅会判断车辆是否能够完成任务并返回仓库，还会在低电量状态下自动触发充电站选择逻辑；在站点选择时，不仅考虑距离，还综合考虑队列长度和站点占用率，从而体现新能源配送场景的现实复杂性。

### 2.4 调度策略、多规模实验与可视化展示

本项目实现了最近任务优先、最大重量优先、最早截止时间优先等多种启发式调度策略，并支持小规模、中规模和大规模等不同场景配置。除此之外，项目进一步尝试将 Q-learning 引入动态调度决策，并构建了离线 MILP 模块用于求解小规模场景的全局方案。基于此项目还通过 Web 前端实现了地图、任务、车辆、充电站和统计指标的实时可视化展示，使系统不仅能够运行仿真，也能够用于演示和结果分析。

Overall 本项目在工程上采用了“仿真引擎 + 算法模块 + 实时接口 + Web 可视化前端”的整体架构。

首先，在底层仿真引擎中，系统负责维护图结构路网、车辆状态、任务状态、充电站状态以及仿真时间推进逻辑。每一个时间步中，系统会依次执行任务释放、任务过期检查、调度决策、车辆移动、电量扣减、充电排队和充电完成等操作，从而构成一个完整的离散时间仿真闭环。

其次，在算法层面，项目将路径规划与任务调度相分离：路径规划负责最短路、目标可达性与能耗可行性分析，调度策略则根据当前可见任务、车辆状态与系统约束选择具体派单动作。这样的模块化设计使得不同调度算法能够在同一仿真环境中公平比较，也便于后续扩展新的路径算法或调度规则。

再次，在进阶方法方面，为响应课程中对强化学习、进阶算法和离线全局优化的鼓励，本项目进一步实现了两类扩展：一类是基于 Q-learning 的超启发式策略选择方法，通过对系统状态进行离散编码，学习在不同场景下应优先采用哪一种底层启发式规则；另一类是基于 MILP 的离线求解模块，在上帝视角下求解小规模实例的全局近优路径与调度方案，并将其结果与在线策略的运行效果进行对比。这使项目不仅停留在“能运行”的层面，也具备了方法对比和分析的研究价值。
最后，在展示层面，系统通过前后端联动实现了实时可视化：后端负责环境状态更新与日志输出，前端负责将地图节点、车辆运动、任务状态变化、充电站队列和统计指标进行图形化展示。用户还可以在界面中切换调度策略、触发离线求解回放以及进行 Q-learning 的训练和推理，从而增强系统的交互性与展示效果。



## 5. 文章结构安排

本文其余部分组织如下：首先介绍系统所采用的核心数据结构与仿真建模方法；随后说明路径规划、调度策略、强化学习与离线优化模块的设计思路；接着展示系统实现与可视化交互方式；最后结合实验结果分析不同策略在新能源物流动态调度任务中的表现，并总结全文工作与未来改进方向。
