# 02 协作接口说明

本页是组内协同开发唯一接口说明。

## 1. 数据流

`panyu.osm.pbf -> processed(parquet/json) -> Environment -> logs`

运行配置流：

`Framework/configs/*.yaml -> run_*.py(--config) -> Environment`

## 2. A 组（算法）接口

策略基类：

- `Framework/scheduler/base.py`
- `select_actions(env) -> list[tuple[vehicle_id, task_id]]`

`Environment` 可调用：

- `get_available_tasks(t=None)`
- `get_idle_vehicle_ids()`
- `get_vehicle_state(vehicle_id)`
- `dispatch(vehicle_id, task_id)`
- `step(actions=None)` / `run(end_time=None, scheduler=None)`
- `get_state_snapshot(raw=False)`
- `get_serializable_snapshot()`
- `export_state_snapshot_json(path)`

说明：默认使用 `raw=False` 快照，避免误改内核对象。

## 3. C 组（可视化/报告）接口

日志导出：

- `Environment.export_logs(output_dir)`

日志文件：

- `step_log.csv/json`
- `vehicle_log.csv/json`
- `task_log.csv/json`
- `station_log.csv/json`
- `events.csv/json`

`run()` 摘要指标：

- `time, total_score, completed, expired, pending, assigned`
- `run_steps, total_distance, avg_vehicle_utilization`
- `avg_station_queue_length, avg_task_completion_time, avg_charge_waiting_time, charge_sessions`

## 4. 协作边界

- A 组不改 `core/simulation.py` 状态机。
- C 组不直接依赖内部对象，统一读日志。
- B 组改字段时遵循“只增不减”，并同步本页文档。

## 5. YAML 约定

- 运行脚本：`run_baseline.py / run_panyu_baseline.py / run_panyu_processed_baseline.py / run_experiment_matrix.py`
- 都支持：`--config <yaml路径>`
- YAML 键名必须用 argparse 的 `snake_case` 参数名
- CLI 参数优先级高于 YAML（可临时覆盖）
