## 3. 当前仍建议在报告里交代的边界（非“未完成核心”，而是诚实范围）

1) **Q-learning 使用前提**  
需存在已训练（或提供的）`q_table.json`；否则 API 选用 `q_learning` 会报错。训练可通过 API 或 `policy/gymnasium_qlearning/train_q_learning.py` 完成。

2) **离线 MILP 的规模与假设**  
`god_view_milp.py` 面向小规模可解释 baseline；规模增大时变量与约束增长快，与动态在线策略的对比更适合在固定小规模实例上说明“差距及原因”。

3) **类型系统中的遗留策略名**  
`UI/logistics-ui/app/types/index.ts` 仍保留 `highest_reward` / `balanced` / `collaborative` 等联合类型成员（标注为本地演示）；**连接远程引擎时**，主推界面只使用与 `server.py` 中 `_map_strategy` 对齐的四种任务策略；若仅本地 `simulation.ts` 演示，行为以前端实现为准。
